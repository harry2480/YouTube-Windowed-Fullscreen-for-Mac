# ソフトウェア製品要件定義書 (PRD)

## 1. プロジェクト概要

- **製品名:** YouTube Windowed Fullscreen for Mac
- **目的:** macOSにおいてYouTubeの「f」キー押下時に発生する「新しい操作スペース（Space）への強制移動」を阻止し、現在のブラウザウィンドウ内のみで全画面表示を実現する。
- **ターゲット:** YouTubeを視聴しながら他のアプリやウィンドウを同一Space内で並行利用したいmacOSユーザー。

## 2. 技術スタック

- **Runtime:** Chromium-based Browser (Chrome, Edge, Brave etc.)
- **Manifest:** Version 3 (MV3)
- **Language:** TypeScript 5.x
- **Build Tool:** Vite 6.x + @crxjs/vite-plugin
- **Styling:** CSS Modules / Vanilla CSS

## 3. 機能要件

### 3.1 キーイベント・インターセプト機能

- YouTubeのプレーヤー上で「f」キーが押下された際、ブラウザ標準のフルスクリーンAPI（requestFullscreen）の発火を即座に停止する。
- 検索ボックス、コメント欄、チャット欄などの入力フォーム（input, textarea, contenteditable）にフォーカスがある場合は、インターセプトを無効化し通常の文字入力を優先する。

### 3.2 擬似フルスクリーン表示機能

- インターセプト成功時、動画プレーヤー要素（#movie_player）に対して独自のCSSクラスを注入する。
- クラス適用時は以下のスタイルを強制する：
  - `position: fixed !important;`
  - `top: 0; left: 0; width: 100vw; height: 100vh;`
  - `z-index: 2147483647;` (符号付き32bit整数の最大値)
- 背景を黒（#000）で塗りつぶし、動画以外のページ要素を完全に隠蔽する。
- ブラウザのスクロールバーを一時的に `overflow: hidden;` に設定し、スクロールを抑止する。

### 3.3 状態トグル機能

- 「f」キーの再押下、または「Esc」キーの押下により、擬似フルスクリーン状態を解除し元のレイアウトに戻す。
- 拡張機能のポップアップUIから、本機能の有効/無効をグローバルに切り替え可能とする。

## 4. 非機能要件

- **パフォーマンス:** スクリプトの実行遅延を10ms以下に抑え、動画のコマ落ちを発生させない。
- **プライバシー:** ユーザーの視聴履歴、個人情報を一切収集・外部送信しない。
- **堅牢性:** YouTubeのUIアップデート（クラス名の変更など）に柔軟に対応できるよう、セレクタは定数管理する。

## 5. UI/UX仕様

- **ポップアップ画面:** - ON/OFFトグルスイッチ。
  - 現在のバージョンの表示。
- **視覚効果:** - macOS標準の「スライド移動」を完全に排除し、瞬時の切り替えを実現する。

## 6. テスト項目

- [ ] 「f」キー押下時にSpaceが追加されないか。
- [ ] シアターモード、ミニプレーヤーモードからの遷移が正常か。
- [ ] ライブ配信のチャット入力中に「f」を打っても全画面化しないか。
- [ ] マルチディスプレイ環境で、対象のブラウザがある画面内のみで全画面化されるか。
